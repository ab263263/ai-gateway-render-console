# 号池 / 中转池控制台实施计划（阶段一）

## 当前阶段
从“测试页 + 手工虚拟模型配置”升级为“测试结果驱动虚拟大模型配置 / 快速组池”。

## 本阶段三块主线

### 1. 测试结果状态标签
目标：平台模型测试后，给每个模型生成可视状态。

状态建议：
- 可用（available）
- 映射异常（mapped_model_mismatch）
- 冷却中（cooldown）
- 不可用（unavailable）
- 未测试（untested）

落地方式：
- 前端先基于当前会话内测试结果做状态映射
- 后续再决定是否持久化到本地数据库

### 2. 虚拟大模型页快速组池入口
目标：不再手工逐个挂 backend，而是从“平台 + 模型状态列表”直接选。

第一版交互：
- 选择平台
- 展示该平台远程模型或已导入模型
- 显示测试状态标签
- 多选模型
- 一键加入到当前虚拟模型或新建虚拟模型
- 负载策略默认 RoundRobin

### 3. 从测试成功模型直接生成 backend 列表
目标：把测试结果直接转成虚拟大模型后端配置。

第一版策略：
- 仅允许“可用 / 映射异常”模型进入候选池
- “冷却中 / 不可用”模型默认禁选或警告
- 自动生成：
  - platform_id
  - model_id
  - weight=1
  - priority=0
  - capabilities（若已有则带上）

## 当前实现顺序
1. 前端测试结果状态映射层
2. 虚拟大模型页快速组池 UI
3. 复用现有 createRoute/addBackend 直接生成 backend 列表
4. 线上部署验证

## 验收标准
- 测试后的模型在 UI 中能看到状态标签
- 虚拟大模型页能按状态选择多个模型
- 一键组池后，后台真实生成 route/backends
- 生成后的虚拟大模型可立即在 `/v1/models` 中可见并可调用
